## New Section: Commit Guidelines

**Requirement**: Before every commit, write documentation in `docs/` directory.

**Purpose**: 
- Ensure changes are documented for future reference
- Help other developers understand what was changed and why
- Maintain project knowledge over time

**Structure**:
```
docs/
├── features/       # Feature documentation
├── changes/        # Change logs
└── setup/          # Setup and configuration notes
```

**Documentation should include**:
1. What was changed
2. Why it was changed
3. Important implementation details
4. Any breaking changes or considerations
